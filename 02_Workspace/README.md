# 🧠 TG-Kim-s-Wiki (P-Reinforce Knowledge Base)

이 저장소는 **P-Reinforce (The Autonomous Gardener)** 에이전트에 의해 자동으로 관리되는 개인 및 가족의 영속적 위키(Persistent Wiki) 아카이브입니다.
파편화된 원시 데이터(Raw Data)를 AI 에이전트가 분석하여 의미론적 카테고리로 분류하고, 지식 간의 유기적인 연결망(Graph)을 스스로 구축합니다.

## 🌟 시스템 개요 (Architecture)
이 시스템은 Andre Karpathy의 LLM-Wiki 아키텍처에서 영감을 받아 구축되었으며, 지식을 수동으로 정리하는 수고를 덜고 인간의 개입을 최소화하면서 지식을 진화시키는 **'지식 자동화 엔진'**을 목표로 합니다.

에이전트는 사용자의 칭찬이나 수정 지시 등 피드백을 수집하여 분류 기준이 되는 정책(Policy)을 지속적으로 업데이트(강화학습)하며, 지식의 타임라인을 보존하기 위해 자동으로 GitHub에 버전을 기록합니다.

## 📂 디렉터리 구조 (Directory Structure)

위키 엔진은 엄격하게 통제되는 3계층 구조로 데이터를 관리합니다.

```plaintext
root/
├── 00_Raw/                 # [불변] 사용자로부터 입력된 가공되지 않은 모든 데이터 보관 (Source of Truth)
│
├── 10_Wiki/                # [자동 구조화] 에이전트가 RL 정책에 따라 분류하고 관리하는 지식 층
│   ├── 🛠️ Projects/        # 목표 중심 (현재 진행 중인 프로젝트, 로드맵 요약)
│   ├── 💡 Topics/          # 개념 중심 (심리학, 코딩, 철학 등 에이전트가 도출한 분류)
│   ├── ⚖️ Decisions/       # 의사결정 중심 (왜 이런 판단을 했는지에 대한 히스토리)
│   └── 🚀 Skills/          # 실행 중심 (유용한 프롬프트, 자동화 워크플로우 패턴)
│
├── 20_Meta/                # [시스템] 지식 엔진의 두뇌 (시스템 운영 데이터)
│   ├── Graph.json          # 지식 간 연결 관계 데이터 (시각화용)
│   ├── Policy.md           # 사용자 피드백이 반영된 분류 정책 및 RL 가중치
│   └── Index.md            # 위키 전체의 입구 (Master Table of Contents)
```

## 📝 지식 변환 규격 (The Wiki Template)
`00_Raw`에 입력된 모든 거친 아이디어는 다음과 같은 양식으로 정제되어 `10_Wiki` 공간으로 재배치됩니다.
- **한 줄 통찰 (The Karpathy Summary)**: 지식의 핵심을 꿰뚫는 단 한 문장
- **구조화된 지식 (Synthesized Content)**: 흩어진 정보에서 추출된 패턴과 세부 내용
- **모순 및 업데이트 (Contradictions & RL Update)**: 과거 지식과의 충돌점 및 정책 변화 기록
- **지식 연결 (Graph)**: 상위 개념, 연관 지식, 원본 문서(`00_Raw`)로 이어지는 쌍방향 링크

## ⚙️ 활용 및 상호작용 (How to interact)
1. 새로운 메모, 아이디어, 스크랩한 글을 `00_Raw/` 에 파일로 던져둡니다.
2. P-Reinforce 에이전트가 이를 스캔하고 내용을 이해한 뒤 적절한 폴더를 찾아냅니다(필요시 폴더를 새로 만듭니다).
3. `10_Wiki/` 에 완벽하게 정리된 마크다운 문서가 생성되고 지식 그래프가 업데이트됩니다.
4. 에이전트가 분류를 잘못했다면 사용자가 "이건 A가 아니라 B로 옮겨줘"라고 지시합니다. 에이전트는 이를 학습하고 다음부터 더 똑똑하게 분류합니다!
