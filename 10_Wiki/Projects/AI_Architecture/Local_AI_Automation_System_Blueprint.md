# [[Local AI Automation System Blueprint]]

## 📌 한 줄 통찰 (The Karpathy Summary)
> This document outlines a plan for building a secure, decentralized, and localized AI automation system connecting professional and personal knowledge graphs.

## 📖 구조화된 지식 (Synthesized Content)
- **추출된 패턴:** The optimal architecture prioritizes data sovereignty, necessitating a hub-and-spoke model where Obsidian acts as the central knowledge graph, and all AI processing occurs via closed-loop, local resources (e.g., Ollama).
- **세부 내용:**
- **Enterprise AI:** Implement a 'Folder Watchdog' agent to automatically parse and structure data (PDFs, documents) added to specific directories.
- **Enterprise Tooling:** Build internal web applications (using Streamlit etc.) to replace standard forms, managing internal resources while maintaining security.
- **Personal Dataflow:** Use Obsidian as the core 'Second Brain' knowledge hub, integrating inputs from mobile and PC devices via structured markdown.
- **Automated Ingestion:** Develop PC-based scripts to collect, summarize, and auto-save recurring data (e.g., investment reports) directly into Obsidian.
- **Notification Layer:** Establish real-time webhooks (e.g., Telegram) to push critical summary data and alerts generated by the automated system.
