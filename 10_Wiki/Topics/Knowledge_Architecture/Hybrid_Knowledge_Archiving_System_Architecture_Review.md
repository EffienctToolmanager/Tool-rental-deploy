# [[Hybrid Knowledge Archiving System Architecture Review]]

## 📌 한 줄 통찰 (The Karpathy Summary)
> The system optimizes data flow by defining specialized roles for multiple repositories (Local PC, GitHub, Google Drive, Obsidian) to balance processing speed with maximum accessibility.

## 📖 구조화된 지식 (Synthesized Content)
- **추출된 패턴:** Effective knowledge management requires distinguishing between the 'High-Speed Processing Zone' (Local PC) and the 'High-Accessibility Zone' (Cloud Sync/Drive).
- **세부 내용:**
- Local PC (C:) is designated as the core workspace for AI agent interaction and rapid development.
- GitHub serves as the primary cloud version control and code backup repository.
- Google Drive (G:) is optimized for final, mobile-accessible knowledge sharing (family/mobile use).
- Obsidian acts as the structured, visual interface for knowledge interconnection.
- The planned pipeline moves data sequentially: Raw Input -> Local Analysis (P-Reinforce Agent) -> Staging Wiki Format -> Cloud Sync (Meta Manager).
